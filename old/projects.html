Raspberry Pi Webserver und Gameserver Projekt

Als ich vor einigen Jahren das erste mal vom Raspberry Pi in einem Youtube Video gehört habe war ich echt begeistert. 2019 kam dann der Raspberry Pi 4 raus, der in der 4GB RAM Variante echt Leistungsstark wurde.

Als erstes Projekt wollte ich einen Minecraft Gameserver hosten. Die Vorteile des Pi's waren dabei offensichtlich: Geringer Stromverbrauch, kein Lüfter, klein und kompakt. Doch hat der neue Pi genug Leistung für einen Vanilla Server? Die Kurze Antwort: Ja hat er.
Im ersten Versuch habe ich einen Cuberite Server gehostet. Dieser Server Client ist in C++ geschrieben und verbraucht daher deutlich weniger Ressourcen als der Vanilla Java Client. Selbst mit 7 Spielern gab es keine Probleme, wenn man nicht gerade in sämtliche Richtungen gleichzeitig fliegt. Doch der Client war mir ein wenig zu buggy.

Der zweite Versuch war der PaperMC Java Server client. Dieser ist zwar deutlich ressourcenfressender als der Cuberite Server, jedoch deutlich sparsamer als der Standard Minecraft Server Client. Damit hat es super geklappt. Es macht mit ca. 5 Leuten enorm laune zu spielen. Allerdings sollte man nicht zu häufig neue Chunks laden, sonst stürzt der Server relativ schnell ab. Trotzdem habe ich nun einen voll funktionsfähigen Minecraft Server im Haus stehen, den ich gemütlich im Regal verstecken und per SSH und Remotedesktop bedienen kann. Es hat sich also gelohnt.

    Dafür wird mein Raspberry Pi benutzt:

  • Webserver mit Nginx
  • Minecraft Gameserver mit dem Paper.MC Java Server Client
  • Cuberite.org Minecraft Gameserver, geschrieben in C++
  • Linux Lernressource für Desktoplose Linux Betriebssysteme
